> 项目路径：`D:\cq\finetune\embedding-finetune`

> 基座模型：[BAAI/bge-small-zh-v1.5](https://huggingface.co/BAAI/bge-small-zh-v1.5)  
> 对比模型：[Qwen/Qwen3-Embedding-0.6B](https://huggingface.co/Qwen/Qwen3-Embedding-0.6B)

## 1. 实验目标

模拟企业内部知识库（电商/FinTech）的语义检索场景，验证：

1. 通用开源 Embedding 在私有数据上的失败模式
2. 小样本领域微调、Hard Negative 微调的效果
3. 1000 条规模数据微调 vs 小样本微调
4. 微调后的小模型 vs 更强的零样本大模型（Qwen3-0.6B）

## 3. 数据说明

### 3.1 知识库语料（Corpus）

- 文件：`data/corpus.json`
- 规模：**25 篇**文档
  - 15 篇主文档（`doc_001` ~ `doc_015`）
  - 10 篇干扰文档（`doc_d1` ~ `doc_d10`，用于制造检索混淆）

### 3.2 测试集

| 名称 | 文件 | 规模 | 用途 |
|------|------|------|------|
| 难例集 | `data/test_cases.json` | **12 条** | 早期手工难例（**与 train_pairs 完全重叠，不能作 Holdout**） |
| 大规模 Benchmark | `data/benchmark_1k.json` | **1000 条** | 与 `train_1k` **100% query 重叠**，仅适合 in-distribution 对比 |
| **未见泛化测试集** | `data/benchmark_500_unseen.json` | **500 条** | **全新 query，与全部训练/测试数据 0 重叠**，真实泛化评测 |

> ⚠️ **关于此前「100% Recall@1」的说明**  
> - `1000条微调` 在 `benchmark_1k` 上评测：训练/测试 query **100% 相同**，97.9% 不能代表泛化。  
> - `12条难例 Holdout`：**12 条 query 全部出现在 train_pairs / train_triplets 中**，100% 属于记忆而非泛化。  
> - 真实泛化请看 **500 条未见测试集**（下文 §6.5）。

### 3.3 训练集

| 名称 | 文件 | 规模 | 说明 |
|------|------|------|------|
| 领域微调 | `data/train_pairs.json` | **25 条** | `(query, doc)` 正样本对 |
| Hard Neg 微调 | `data/train_triplets.json` | **41 条** | 25 正样本对 + 16 三元组（含 hard negative） |
| 1000 条微调 | `data/train_1k.json` | **1000 条** | 由 `benchmark_1k.json` 生成，与大规模测试集 **query 相同** |

### 3.4 典型失败场景（私有数据）

| 类别 | 示例 Query | 难点 |
|------|-----------|------|
| abbreviation | 列表里用户会不会点进去 | 不说 CTR，与 CVR/画像等混淆 |
| colloquial | 买家不想买了怎么把钱退回去 | 口语 vs 正式流程文档 |
| product_code | 7821那台风控盒子是干嘛的 | 内部 SKU 编码关联弱 |
| domain_synonym | 怎么拦住刷单和黑产注册 | 与支付渠道风控语义接近 |
| entity_disambiguation | 线上配置能直接在后台改吗 | 未提 Apollo，与 K8s 发布混淆 |
| domain_jargon | 包裹发到哪了在哪查进度 | OMS vs WMS 物流体系混淆 |

## 4. 模型与训练方案

### 4.1 参与对比的模型

| 代号 | 模型路径 | 训练数据 | 参数量 |
|------|----------|----------|--------|
| 基线 | `BAAI/bge-small-zh-v1.5` | 无 | 24M |
| 领域微调 | `models/bge-small-zh-domain` | 25 条 | 24M |
| Hard Neg 微调 | `models/bge-small-zh-hard-neg` | 41 条（在领域模型上继续） | 24M |
| 1000 条微调 | `models/bge-small-zh-1k` | 1000 条（从基线重训） | 24M |
| Qwen3 | `Qwen/Qwen3-Embedding-0.6B` | 无（零样本） | 596M |

### 4.2 训练配置

| 阶段 | 损失函数 | Epochs | Batch | LR | 基座 |
|------|----------|--------|-------|-----|------|
| 领域微调 | MultipleNegativesRankingLoss | 4 | 8 | 2e-5 | BGE-small 基线 |
| Hard Neg 微调 | MultipleNegativesRankingLoss | 6 | 8 | 1e-5 | 领域微调模型 |
| 1000 条微调 | MultipleNegativesRankingLoss | 3 | 32 | 2e-5 | BGE-small 基线 |

**Hard Negative 来源**：基线 / 普通微调在失败用例上的误召回文档（Top1 + Top3 干扰项）。

### 4.3 关键概念速查

| 概念 | 说明 |
|------|------|
| **MNRL 损失** | 对比学习：拉近 `(anchor, positive)`，推远 batch 内其他 positive 及显式 hard neg |
| **In-batch negatives** | 同 batch 其他样本的 positive 自动作为负样本 |
| **Hard negatives** | 模型易混淆的误召回文档，显式作为第三列负样本 |
| **warmup_steps** | 学习率从 0 线性升至峰值再衰减，防止破坏预训练权重 |
| **Qwen3 query prompt** | query 编码需 `prompt_name="query"`，document 不加 prompt |

## 5. 评测协议（标准化、可对比）

| 项目 | 统一设置 |
|------|----------|
| 相似度 | Cosine（向量 L2 归一化） |
| Batch size | 32 |
| Warmup | 10 条 query（计时前） |
| 设备 | CPU |
| 指标 | Recall@1/3/5/10、MRR、NDCG@10、延迟、QPS、模型大小 |

> **评测口径说明**  
> - **1000 条 Benchmark**：五模型在同一 1000 条 query 上评测；其中「1000 条微调」模型训练数据与测试数据 **重叠**，指标偏高（in-distribution）。  
> - **12 条 Holdout**：独立难例，**未参与任何训练**，更能反映真实泛化。

### 6.5 真实泛化：500 条未见测试集（推荐参考）

- 文件：`data/benchmark_500_unseen.json`
- 与现有 **1016 条** query（train + benchmark_1k + test_cases）**零重叠**
- **所有模型均未在此 500 条上训练**
- 复现：`python run_eval_unseen_500.py`

#### 检索效果（500 queries, CPU）

| 模型 | 训练样本 | Recall@1 | Recall@3 | Recall@5 | MRR | NDCG@10 | 失败@1 |
|------|----------|----------|----------|----------|-----|---------|--------|
| BGE-small 基线 | 0 | 76.4% | 89.8% | 93.8% | 0.842 | 0.872 | 118 |
| 25 条领域微调 | 25 | 82.0% | 94.8% | 96.6% | 0.885 | 0.910 | 90 |
| 41 条 Hard Neg | 41 | 83.0% | 95.0% | 96.6% | 0.893 | 0.917 | 85 |
| 1000 条微调 | 1000 | 84.6% | 93.6% | 95.8% | 0.897 | 0.919 | 77 |
| **Qwen3-0.6B** | 0 | **88.6%** | **98.2%** | **100%** | **0.934** | **0.951** | **57** |

#### 泛化场景下的结论

1. **此前 100% 确有过拟合/泄漏问题**；在未见 500 条上，最好模型为 **Qwen3 88.6%**，非 100%。
2. **微调仍然有效**（相对基线 76.4% / 89.8% / 93.8%）：
   - 1000 条微调：R@1 +8.2%，R@3 +3.8%，R@5 +2.0%
   - 41 条 Hard Neg：R@1 +6.6%，R@3 +5.2%，R@5 +2.8%
3. **Qwen3 在 R@1/R@3/R@5 均领先**：88.6% / 98.2% / 100% vs 微调最好 84.6% / 93.6% / 95.8%
4. **小样本 Hard Neg 依然划算**：41 条训练接近 1000 条微调效果（83.0% vs 84.6%），仅差 1.6%。

报告：`results/benchmark_500_unseen_report.json` / `.csv`

## 7. 结论与选型建议

### 7.1 核心结论

1. **此前 100% R@1 存在训练-测试重叠，不能代表泛化**（1000 条全重叠；12 条难例全在 train_pairs 中）。
2. **500 条未见测试集上**：Qwen3 零样本 **88.6%** 最高；1000 条微调 **84.6%**；基线 **76.4%**。
3. **微调在未见数据上仍有效**：1000 条 +8.2%、41 条 Hard Neg +6.6% vs 基线。
4. **1000 条微调未超越 Qwen3（未见集）**：说明大数据微调提升泛化，但强通用模型零样本仍有优势。
5. **小样本 Hard Neg 性价比突出**：41 条 ≈ 84% 效果，接近 1000 条的 84.6%。

### 7.2 场景选型

| 场景 | 推荐方案 |
|------|----------|
| 无标注、有 GPU、追求最高泛化 | Qwen3-Embedding-0.6B（未见集 88.6%） |
| 私有知识库、CPU 部署、低延迟 | 41 条 Hard Neg 或 1000 条微调 BGE-small |
| 标注极少（<50 条） | 领域微调 + Hard Neg 两阶段 |
| 有大量 query-doc 日志（数百~千条） | 1000 条规模 MNRL 微调 |
| 快速验证 | BGE-small 基线 |

### 7.3 注意事项

- 1000 条微调在相同 1000 条上评测会有 **训练-测试重叠**，97.9% 偏乐观；务必辅以 Holdout 或划分 train/test。
- Qwen3 需 Python 3.9+ 和 `transformers>=4.51`，与 Python 3.8 环境不兼容。
- 国内环境建议设置 `HF_ENDPOINT=https://hf-mirror.com`。
